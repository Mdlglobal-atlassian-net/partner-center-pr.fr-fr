## <a href="" id="processfaq"></a>Processus

**Qu’est le nouveau processus de confirmation d’acceptation du client de l’accord de Cloud de Microsoft ?**

Microsoft s’engage à aider nos clients et nos partenaires à rester en conformité. En tant que partenaire dans le programme de fournisseur de solutions Cloud (CSP) de Microsoft, vous êtes responsable de le conditions examen Microsoft générales, notamment l’utilisation acceptable de produits et services avec les clients Microsoft. Pour mieux aider les partenaires pour répondre aux exigences de conformité et pour assurer une transparence, nous mettons en place un nouveau processus qui nécessite des partenaires CSP confirmer que leurs clients ont accepté les termes du contrat de Cloud de Microsoft. À compter du 7 août 2018, nous avons introduit un nouveau champ dans le tableau de bord Partner Center interface utilisateur également une API espace partenaires pour partenaires pour fournir ces informations. Initialement il sera facultatif pour fournir cette confirmation, mais elle sera nécessaire à partir du 7 novembre 2018. 

**Quand cette exigence n’accèdent pas en vigueur ?**

Les partenaires peuvent commencer à confirmer l'acceptation dès la mise à disposition de cette fonctionnalité, le 7 août 2018. À partir du 7 novembre 2018, le tableau de bord Partner Center les utilisateurs doivent confirmer l’acceptation du client de l’accord de Cloud de Microsoft sur toutes les nouvelles transactions. Le 22 mars 2019 utilisateurs de l’API espace partenaires sera nécessaires pour faire de même. La date limite pour les utilisateurs de l’API a été étendue pour leur permettre de supplémentaires temps pour l’implémentation technique.

**NOUVEAU | Microsoft valide l’exactitude des informations client fournies par le partenaire ?**

Microsoft vérifie que l’adresse e-mail du client fournie est de format correct. Il incombe au partenaire pour fournir des informations précises et de corriger toutes les informations qui se trouve être inexactes. 

**NOUVEAU | Comment pouvez un partenaire prouver acceptation du client de l’accord de Cloud Microsoft, s’il est signé, documenté, numérisés, ou sur du papier ?**

Le partenaire est responsable de s’assurer que le client a accepté les termes du contrat de Cloud Microsoft et pour la gestion de leur propre processus et les enregistrements. En utilisant le processus de tableau de bord de centre de partenaires ou des API espace partenaires, le partenaire est de confirmer que le client a accepté les termes du contrat de Cloud de Microsoft.

**NOUVEAU | Comment Microsoft vérifie que le processus de confirmation complète pour un client ?**

À compter le 7 novembre 2018, quand un partenaire CSP démarre une transaction en utilisant le tableau de bord de centre de partenaires pour un client existant ou nouveau, systèmes de Microsoft automatiquement vérifiera si les informations de confirmation du client sont présentes dans l’espace partenaires Tableau de bord. Si tel n’est pas le cas, la transaction ne pourra pas aboutir. 

À compter de mars 22, 2019, quand un partenaire CSP lance une transaction pour un client existant ou nouveau à l’aide de l’API espace partenaires, systèmes de Microsoft vérifie automatiquement si les informations de confirmation du client sont présentes. Si tel n’est pas le cas, la transaction ne pourra pas aboutir. 

**NOUVEAU | Notification est envoyée aux clients dont l’adresse e-mail est fourni dans le processus de confirmation ?**

Il n’existe aucun processus de notification.

## <a href="" id="policyfaq"></a>stratégie

**Pourquoi est-ce que Microsoft nécessitant partenaires confirmer que les clients ont accepté le contrat de Cloud de Microsoft ?** 

L’accord de Cloud de Microsoft protège les partenaires, clients et Microsoft en fournissant des droits d’utilisation du client, termes et conditions accordées pour les commandes de produits et des services de Cloud Microsoft. 

Une exécution correcte de l’accord de Cloud de Microsoft protège de nos partenaires, les clients et les Microsoft en vous assurant d’alignement mutuelle sur nombreux sujets importants, y compris mais pas limité à la sécurité, confidentialité et protection des données. Le processus de confirmation garantit la transparence et l’alignement et permet à Microsoft de mieux aider les clients doivent pouvoir répondre rapidement aux enquêtes réglementaires.

**Les informations de client sont nécessaires pour terminer le processus de confirmation ?**
Les informations nécessaires concernant la personne qui a accepté le contrat sont les suivantes :
-   Date d’acceptation du contrat
-   Nom de famille et prénom
-   Adresse électronique
-   Numéro de téléphone (facultatif) 

**NOUVEAU | Que se passe-t-il si nous ne répondent pas à la date d’échéance ?**

Les partenaires peuvent commencer à confirmer l'acceptation dès la mise à disposition de cette fonctionnalité, le 7 août 2018. À partir du 7 novembre 2018, le tableau de bord Partner Center les utilisateurs doivent confirmer l’acceptation du client de l’accord de Cloud de Microsoft sur toutes les nouvelles transactions. Le 22 mars 2019 utilisateurs de l’API espace partenaires sera nécessaires pour faire de même.

Au moment de l’application, nouveaux achats pour un client existant ou des modifications pour le nombre sur les abonnements existants de sièges ne seront pas possible jusqu'à ce que l’acceptation du client est confirmée. Pour un nouveau client, les transactions ne seront pas autorisées jusqu'à ce que l’acceptation soit confirmée. Le renouvellement automatique n’est pas concerné par cette exigence.

**Quelles régions cette exigence s’applique-t-elle ?**

Confirmation d’acceptation du client à l’accord de Cloud de Microsoft est obligatoire pour toutes les régions dans le Cloud Public Microsoft. La confirmation de l’acceptation du client n’est pas compatible avec les clouds nationaux (notamment Microsoft Cloud for US Government et Microsoft Cloud Germany).

**Les partenaires sont responsables de confirmation d’acceptation du client à l’accord de Cloud de Microsoft ?**

À partir du 7 novembre 2018, les fournisseurs indirects de fournisseur de services Cloud (CSP) et les partenaires CSP électronique sont tous deux responsables de confirmation d’acceptation du client et peut utiliser l’API espace partenaires ou un tableau de bord de centre de partenaires au pour faire. Fournisseurs indirects devrez peut-être commencer par collecter les informations requises à partir de leurs revendeurs indirects et puis rendez-le accessible à Microsoft via l’API espace partenaires ou d’un tableau de bord de centre de partenaires. Engagera, il est le fournisseur Indirect qui a une liaison de contrat avec Microsoft pour vous assurer que les clients finaux de leurs revendeurs CSP ont accepté le contrat de Cloud de Microsoft. 

**Quels scénarios client nécessitent une confirmation d’acceptation du client de l’accord de Cloud de Microsoft ?**

L’acceptation de l’accord de Cloud de Microsoft doit être confirmée lorsque les partenaires CSP veux les transactions suivantes :
1.  Un nouveau client termine nouveaux achats
2.  Un client existant est terminée de nouveaux achats
3.  Un client existant met à jour le nombre de sièges pour les abonnements existants

**Puis-je confirmer l’acceptation des clients existants distincts à partir de leurs nouveaux achats ou ajouts de siège ?**

Oui. Acceptation du client peut être vérifiée pour tous les clients couverts par l’accord de Cloud de Microsoft.

**Quelles sont les exigences des clients qui ont plusieurs partenaires CSP ?**

La confirmation d’acceptation est confirmée par le partenaire. Si un client effectue des transactions avec plusieurs partenaires fournisseurs de solutions Cloud, chaque partenaire doit confirmer l’acceptation.

**Je suis un revendeur CSP Indirect utilisation des fournisseurs indirects de CSP. Ce qui m’appartient ?**

Engagera, il est le fournisseur Indirect qui a une liaison de contrat avec Microsoft pour vous assurer que le client final a accepté le contrat de Cloud de Microsoft. Cependant, les fournisseurs indirects peuvent avoir des accords distincts avec des revendeurs indirects pour recueillir l’acceptation du client. Dans ce cas, les fournisseurs indirects devront peut-être d’abord recueillir les informations d'acceptation du client nécessaires auprès de leurs revendeurs indirects, puis les mettre à disposition de Microsoft via l'API de l'Espace partenaires ou le tableau de bord de l'Espace partenaires. Vous pouvez prendre en charge ce processus en fournissant rapidement les informations requises à vos fournisseurs indirects à leur demande.

**Je suis un fournisseur CSP Indirect. Ai-je besoin pour se coordonner avec mon revendeur CSP Indirect à être préparé ?**

Il est conseillé aux fournisseurs indirects qui doivent recueillir les informations d’acceptation du client requises auprès de leurs revendeurs indirects de communiquer immédiatement cette exigence à leurs revendeurs indirects afin de coordonner la distribution et la réception de ces informations.

**Comment puis-je j’expliquer à mes clients qu’ils doivent confirmer leur acceptation de l’accord de Cloud de Microsoft ?**

Une exécution correcte de l’accord de Cloud de Microsoft protège Microsoft, ses partenaires et les clients en vous assurant d’alignement mutuelle sur nombreux sujets importants, y compris mais pas limité à la sécurité, confidentialité et protection des données. Le processus de confirmation garantit la transparence et l’alignement et permet à Microsoft de mieux aider les clients doivent pouvoir répondre rapidement aux enquêtes réglementaires.

**NOUVEAU | Si un client fonctionne avec un seul fournisseur CSP Indirect, mais les transactions sont terminées avec plusieurs revendeurs indirects de CSP, chaque revendeur CSP Indirect doit-elle fournir le fournisseur CSP Indirect avec les informations de confirmation ?**

Le fournisseur CSP Indirect est chargé de fournir la confirmation d’acceptation du client Microsoft Cloud accord à Microsoft. Pour chaque compte client, confirmation qu’une seule peut être enregistrée dans le tableau de bord de centre de partenaires en même temps. 

Il incombe au fournisseur CSP Indirect pour confirmer qu’une personne à contacter client responsable pour accepter le contrat de Cloud de Microsoft a effectué cette opération, quel que soit le nombre de revendeurs indirects de CSP qui ils transact avec.

**NOUVEAU | Qui doit être la personne autorisée à confirmer l’acceptation de l’accord de Cloud de Microsoft à l’intérieur de l’organisation du client ?**

Il incombe au partenaire pour obtenir l’acceptation du client de l’accord de Cloud Microsoft à partir d’une personne à l’intérieur de l’organisation du client. Il est la responsabilité du client pour déterminer qui est autorisé à fournir d’acceptation. 

**NOUVEAU | Quelle version de l’accord de Cloud de Microsoft sont des partenaires responsables de confirmation d’acceptation du client ? Partenaires doivent indiquer que la version de l’accord de Cloud de Microsoft a été acceptée par le client ?**

Partenaires sont responsables d’à l’aide de l’accord de Cloud de Microsoft plus récente lors de l’établissement des transactions CSP avec un nouveau client. Les partenaires doivent confirmer l’acceptation de l’accord de Cloud de Microsoft plus récente lors du renouvellement, la modification ou la fin de nouvelles transactions avec un client existant. 

Dans le cas que l’accord de Cloud de Microsoft est mis à jour avant le renouvellement, partenaires ne sont pas obligés d’avoir au client de ré-obtenir immédiatement acceptation du client de l’accord de Cloud de Microsoft, mais doivent confirmer à nouveau son consentement à l’avant de l’accord du Cloud Microsoft à renouveler. Toutefois, vous pouvez souhaiter demandez au client d’accepter les nouvelles conditions d’utilisation avant le renouvellement pour tirer parti des nouvelles et améliorées de termes dans le contrat de Cloud de Microsoft.

**NOUVEAU | Je suis un revendeur CSP Indirect. Si j’enregistre l’acceptation du client de l’accord de Cloud de Microsoft dans les enregistrements de mon entreprise, dois-je également fournir ces informations via le tableau de bord de centre de partenaires ou l’API espace partenaires ?**

Engagera, il est le fournisseur Indirect qui a une liaison de contrat avec Microsoft pour vous assurer que le client final a accepté le contrat de Cloud de Microsoft. Cependant, les fournisseurs indirects peuvent avoir des accords distincts avec des revendeurs indirects pour recueillir l’acceptation du client. Dans ce cas, les fournisseurs indirects devront peut-être d’abord recueillir les informations d'acceptation du client nécessaires auprès de leurs revendeurs indirects, puis les mettre à disposition de Microsoft via l'API de l'Espace partenaires ou le tableau de bord de l'Espace partenaires.

**NOUVEAU | Que se passe-t-il si mon client n’accepte pas les termes du contrat de Cloud de Microsoft ?**

En tant que partenaire dans CSP programme Microsoft, il vous incombe d’acceptation du client de l’accord de Cloud de Microsoft avant de passer une commande. Cela n’est pas une nouvelle spécification. Pour une raison quelconque, si le client accepte n’est plus à l’accord de Cloud Microsoft, puis le partenaire doit annuler les abonnements associés à ce client.

**NOUVEAU | Un revendeur peut fournir des consentement au nom du client ?**

Non.

**Une fois que je confirme qu’un client a fourni l’acceptation, dois-je reconfirmer il pour le client à l’avenir ?**

Une fois que vous avez confirmé que le client a accepté le contrat de Cloud Microsoft, ces informations restent dans la section compte client espace partenaires, et il n’est pas nécessaire de reconfirmer l’acceptation de la durée de temps que le client est couvert par les termes du contrat. Vous devez uniquement reconfirmer acceptation avant de le placer un nouvel ordre.

Dans le cas que l’accord de Cloud de Microsoft est mis à jour avant le renouvellement, partenaires ne sont pas obligés d’avoir au client de ré-obtenir immédiatement acceptation du client de l’accord de Cloud de Microsoft, mais doivent confirmer à nouveau son consentement à l’avant de l’accord du Cloud Microsoft à renouveler. Toutefois, vous pouvez souhaiter demandez au client d’accepter les nouvelles conditions d’utilisation avant le renouvellement pour tirer parti des nouvelles et améliorées de termes dans le contrat de Cloud de Microsoft.

**NOUVEAU | Je ne suis pas à l’aise donnant des informations de contact direct de mon client final de Microsoft. Comment Microsoft utilise ces informations ?**

Lorsqu’un partenaire crée un locataire client, il est nécessaire pour fournir des informations de contact principales pour le client. Par les termes du contrat de Services en ligne, « Microsoft traite les données personnelles uniquement sur les instructions relatives à partir du client. »

## <a href="" id="technicalfaq"></a>Techniques

**Comment les partenaires confirmer l’acceptation du client et fournir à Microsoft ? Quels outils sont disponibles pour prendre en charge ce processus ?**

Il existe deux méthodes sont disponibles à partir du 7 août 2018 pour aider les partenaires : 
1.  Les partenaires peuvent utiliser l'API de l'Espace partenaires pour confirmer par programme l’acceptation du client. 
2.  Partenaires peuvent utiliser le tableau de bord de centre de partenaires pour confirmer l’acceptation du client.

**NOUVEAU | Je suis un partenaire CSP qui utilise l’API espace partenaires. Les modifications de l’API ai-je besoin pour prendre en compte ?** 

Dans ce cas, les modifications de l’API pour prendre en compte sont :

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/get-confirmation-of-customer-consent) 

-   [https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata](https://docs.microsoft.com/en-us/partner-center/develop/get-agreement-metadata)

-   [https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent](https://docs.microsoft.com/en-us/partner-center/develop/confirm-customer-consent)

**Que se passe-t-il si je rencontre des difficultés de mise en œuvre de l’API espace partenaires ?**

L'acceptation peut être confirmée manuellement via le tableau de bord de l'Espace partenaires.

**NOUVEAU | Comme alternative à l’utilisation du tableau de bord de centre de partenaire ou l’API espace partenaires, puis-je en bloc télécharger un fichier avec les informations de confirmation de client à la place ?**

À ce stade, le tableau de bord de centre de partenaires et de l’API espace partenaires sont le seul moyen accepté de fournir la confirmation d’acceptation du client de l’accord de Cloud de Microsoft.

**NOUVEAU | Je suis un partenaire à l’aide d’une plateforme de tiers, ou « middleware », qui effectue des fonctions pour connecter des systèmes de ma société avec l’espace partenaires. Que dois-je faire ?**

Partenaires à l’aide de plateformes qui s’intègrent l’API espace partenaires sont invités à entretenez-vous directement avec les fournisseurs concernant leurs plans et les chronologies pour intégrer les fonctionnalités de l’API espace partenaires qui automatise le processus de confirmation du client.

**NOUVEAU | Pouvez-vous fournir un exemple de flux d’API pour l’importation du nom complet du client, une adresse e-mail et une date d’acceptation du contrat de Cloud de Microsoft ?**

Les partenaires sont invités à consulter la documentation de l’API espace partenaires pour plus d’informations et des exemples.

**NOUVEAU | Comment obtenir l’accord de Cloud de Microsoft plus récente à afficher au client et à utiliser dans l’enregistrement de confirmation dans le cadre de mon processus de l’API ?**

L’API espace partenaires en cours est très élémentaire et ne fournit pas pour les partenaires récupérer par programme une copie de l’accord de Cloud Microsoft, ni il capture les versions de l’accord de Cloud de Microsoft acceptés par un client dans le cadre de la confirmation. 

**NOUVEAU | À l’aide de l’API espace partenaires pour ce processus requiert un paramètre de nom d’utilisateur. Comment gérer cette exigence lorsque les commandes sont passées via des plateformes de fournisseurs tiers qui ne s’intègrent avec Azure AD ?**

Dans ce cas, l’ID d’utilisateur identifie l’employé dans le client partenaire qui fournit la confirmation de consentement du client. Si l’employé n’a pas d’un compte d’utilisateur Azure AD correspondant sous le locataire du partenaire, nous vous recommandons de désigner un compte d’utilisateur spécifique à cet effet et créer un processus de capture séparément de l’identité de l’employé en fournissant le confirmation. 

**NOUVEAU | Pourquoi est-ce que Microsoft nécessitant le partenaire confirmer le consentement du client à l’accord de Cloud de Microsoft au lieu d’obtenir directement à partir du client ?**

Le client achète auprès du partenaire CSP, pas directement auprès de Microsoft. Pour l’utilisation des services que Microsoft fournit, Microsoft fournit des conditions d’utilisation qui sont documentées dans l’accord de Cloud de Microsoft. Le contrat de revendeur Cloud Microsoft nécessite que les clients Lisez et acceptez le contrat de Cloud de Microsoft avant de passer une commande, les partenaires CSP. 

**NOUVEAU | J’utilise l’interface d’utilisateur web API et le tableau de bord de centre de partenaires pour confirmer l’acceptation du client accord du Cloud Microsoft. Après le 7 novembre 2018, quelles sont mes options pour confirmer l’acceptation du client ?**

Les deux options sont disponibles et acceptable de confirmation d’acceptation du contrat de Cloud de Microsoft client. Transactions à l’aide du tableau de bord de centre de partenaires seront nécessitent une confirmation à être présent à compter du 7 novembre 2018. Transactions à l’aide de l’API espace partenaires seront nécessitent une confirmation à être présent à compter du 22 mars 2019.
